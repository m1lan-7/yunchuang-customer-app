# 云创客户跟进 App

本应用用于温州云创一级/二级客户的跟进管理。当前版本不实时读取飞书原表，数据以每天手动复制粘贴到 App 为准。

## 打开地址

```text
http://127.0.0.1:4184
```

团队共享版部署后，会变为 HTTPS 地址，并可配置到飞书工作台网页应用入口。部署说明见 `DEPLOYMENT.md`。

## 数据更新方式

1. 在页面右侧「台账更新」中选择一级或二级。
2. 从 Excel/飞书表格复制表头和客户数据。
3. 粘贴到输入框，点击保存当前台账。

建议二级客户表头包含：

```text
客户姓名、联系方式、渠道归属、模块、评级、到访日期、获客端口、客户情况、预计下次到访时间
```

建议一级客户表头包含：

```text
客户姓名、联系方式、系统归属、模块、获客日期、预计来访时间、获客端口、客户情况、是否来访、是否成交
```

## 跟进规则

- 一级、二级客户严格分开统计和管理。
- 二级客户评级中 `c` 会自动归并为 `C`。
- 获客模块按 6 月月报人员分组重算为四类：乘风破局、花开富贵、销冠特工、KA。
- 每个客户可点击「跟进」，填写今日是否到访、最新情况、下次预计到访时间。
- 「预计到访客户名单」可选择今日、明日、本周。
- 顶部统计周期默认本月，可切换某月、整年或全部。
- KPI、四组客户池、人员客户池、C级客户名单均可点击下钻到客户明细。
- KPI、四组客户池点击后以弹窗展示客户名单，不改变底部主明细表。
- 人员客户池点击后以弹窗同时展示该人员一级和二级客户。
- 人员卡片展示所属小组、一级量、二级量、C级量，并自动标记低活跃、一转二低、C级待铆定、跟进不足等标签。
- 人员客户池和C级客户名单均支持按小组筛选，C级名单还支持按人员筛选。
- A级代表已成交客户；C级代表到访且意向度高客户；D级代表到访但意向度不高客户。

## 数据文件

客户数据保存在本机：

```text
data/customers.json
```

该文件只在本机更新，不会写回飞书。

## 团队共享部署

当前代码支持两种运行模式：

- 本机模式：不配置 `DATABASE_URL`，继续使用 `data/customers.json`。
- 云端模式：配置 `DATABASE_URL` 后使用 PostgreSQL 保存数据。

设置 `APP_PASSWORD` 后会启用统一密码登录，适合第一版团队共享使用。
